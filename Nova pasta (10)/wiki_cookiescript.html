Introdução

CookieScript é uma linguagem de script híbrida que combina operações de sistema (Python) com operações web (JavaScript). Desenvolvida para facilitar a criação de ferramentas que precisam interagir tanto com o sistema operacional quanto com serviços web.

O projeto permite escrever scripts simples que podem executar tarefas como manipulação de arquivos, requisições HTTP, criptografia, operações matemáticas, manipulação de strings, acesso a bancos de dados e muito mais, tudo em uma sintaxe unificada.

Atualmente conta com 19 módulos integrados cobrindo desde operações básicas do sistema até funcionalidades avançadas como processamento de dados, comunicações, monitoramento do sistema, incorporação de HTML e criação de serviços web compiláveis. Suporta também definição de funções personalizadas, estruturas de controle (if/else, for, while) e tratamento de erros (try/catch).

Instalação

Download

Pré-requisitos

  • Python 3.8+

Passos de Instalação

  1. Extraia o arquivo ZIP baixado
  2. Abra o terminal na pasta extraída
  3. Instale dependências Python:
    pip install -r requirements.txt
  4. Instale dependências Node.js:
    npm install

Configuração Local

Se você já tem o projeto baixado localmente:

  1. Navegue até a pasta do projeto
  2. Ative o ambiente virtual: .\.venv\Scripts\activate (Windows)
  3. Instale dependências: pip install -r requirements.txt
  4. Para Node.js: npm install

Para executar um script CookieScript:

Ou interativamente:

python run_cookiescript.py

Linguagem CookieScript

Sintaxe Básica

CookieScript suporta tipos dinâmicos com verificação automática:

  • int: Números inteiros (42, -10)
  • float: Números decimais (3.14, -2.5)
  • str: Texto ("Olá", 'Mundo')
  • bool: Valores booleanos (true, false)
  • list: Listas ([1, 2, 3])
  • dict: Dicionários ({chave: valor})

O sistema verifica automaticamente a compatibilidade de tipos em operações:

// Operações válidas numero = 42 + 8 // int + int = int decimal = 3.14 + 1.86 // float + float = float texto = "Olá" + " Mundo" // str + str = str // Conversões automáticas numero = 123 + "456" // int + str = "123456" (concatenação)

Verificação de Tipos

O sistema de tipos previne operações inválidas:

// Operações permitidas if (numero > 10) { ... } // Comparação numérica if (texto == "teste") { ... } // Comparação de strings // Operações bloqueadas (geram erro) numero + lista // Erro: tipos incompatíveis booleano - string // Erro: operação não suportada

Tratamento de Erros de Tipo

Erros de tipo são reportados claramente:

// Exemplo de erro resultado = 123 + [1, 2, 3] // Erro: Operação '+' não suportada entre int e list

Matriz de Compatibilidade de Tipos

Operação Tipos Compatíveis Resultado
+ int + int, float + float, str + str, int + str, float + str número ou string
- int - int, float - float número
* int * int, float * float número
/ int / int, float / float float
== qualquer combinação do mesmo tipo bool
>, <, >=, <= int/int, float/float bool

Variáveis

Variáveis são criadas com o operador =:

resultado = network.http_request(url="https://api.exemplo.com") hash = crypto.hash_sha256(dados=resultado)

Funções Definidas pelo Usuário

Você pode definir suas próprias funções:

// Definir função function saudacao(nome) { resultado = "Olá, " + nome + "!" } // Chamar função saudacao("Mundo") mensagem = resultado

Funções podem ter múltiplos parâmetros e usar estruturas de controle:

function calcular_area(base, altura) { if base > 0 and altura > 0 { area = math.multiplicar(base, altura) } else { area = 0 } } calcular_area(5, 10) resultado_area = area

Estruturas de Controle

CookieScript suporta estruturas condicionais e de loop:

// Condicional if numero > 5 { status = "maior" } else { status = "menor" } // Loop for for item in lista { soma = math.somar(soma, item) } // Loop while contador = 0 while contador < 10 { contador = math.somar(contador, 1) filesystem.escrever_arquivo(caminho="log.txt", conteudo="Contador: " + contador, modo="a") }

Operações com Arquivos

O módulo filesystem permite manipular arquivos com diferentes modos:

// Escrever (sobrescrever arquivo) filesystem.escrever_arquivo(caminho="arquivo.txt", conteudo="Conteúdo", modo="w") // Adicionar ao final do arquivo filesystem.escrever_arquivo(caminho="log.txt", conteudo="Nova linha\n", modo="a") // Ler arquivo conteudo = filesystem.ler_arquivo(caminho="arquivo.txt") // Deletar arquivo filesystem.deletar_arquivo(caminho="arquivo.txt")

Sistema de Sandbox e Proteção

CookieScript inclui um sistema robusto de proteção contra análise e execução em ambientes hostis:

// Verificar se está em debugger debugger_detectado = antidebug.verificar_debugger() // Verificar se está em máquina virtual vm_detectada = antidebug.verificar_vm() // Verificar se está em sandbox sandbox_detectado = antidebug.verificar_sandbox() // Sair se ambiente hostil for detectado antidebug.sair_do_programa() // Forçar modo sandbox restritivo antidebug.forcar_sandbox()

Tratamento de Erros

Use try/catch para tratamento de erros:

try { resultado = math.dividir(10, 0) } catch { erro = "Divisão por zero: " + error_message }

Compatibilidade com Módulos de Outras Linguagens

CookieScript permite a integração com módulos e bibliotecas de outras linguagens de programação, expandindo suas capacidades:

  • Módulos Python: Através do módulo python, você pode importar dinamicamente qualquer módulo Python instalado no sistema. Isso inclui bibliotecas como NumPy, Pandas, Requests, entre outras.
  • Bibliotecas JavaScript: Para operações web, o sistema utiliza WebSocket para comunicar com um cliente JavaScript, permitindo acesso a APIs e bibliotecas do navegador como DOM manipulation, Canvas, Web APIs, etc.
  • Extensibilidade: Novos módulos podem ser criados em Python e facilmente integrados ao sistema, permitindo compatibilidade com praticamente qualquer biblioteca Python existente.
// Exemplo de importação de módulo Python python.importar_modulo(nome="requests") resposta = python.chamar_funcao(modulo="requests", funcao="get", url="https://api.exemplo.com") // Exemplo de uso de NumPy (se instalado) python.importar_modulo(nome="numpy") array = python.chamar_funcao(modulo="numpy", funcao="array", lista=[1, 2, 3, 4, 5]) media = python.chamar_funcao(modulo="numpy", funcao="mean", array=array)

Nota: Para usar módulos Python externos, eles devem estar instalados no ambiente Python do sistema. Use pip install nome_do_modulo para instalar bibliotecas adicionais.

Arquitetura

O CookieScript é composto por:

  • Interpretador Python: Processa os comandos e executa operações do sistema
  • Cliente JavaScript: Executa operações web via WebSocket
  • Servidor WebSocket: Coordena comunicação entre Python e JavaScript
  • Módulos: Classes Python que implementam funcionalidades específicas

Fluxo de Execução

  1. Script é carregado pelo interpretador Python
  2. Cada linha é parseada e executada
  3. Chamadas para módulos são direcionadas para as classes apropriadas
  4. Operações web são enviadas via WebSocket para o cliente JavaScript
  5. Resultados são retornados e processados

FAQ

O que diferencia CookieScript de outras linguagens?

CookieScript é específico para tarefas que requerem integração entre sistema operacional e web services, com foco em automação e ferramentas de administração.

É seguro usar CookieScript?

Como qualquer ferramenta de automação, depende do uso. O projeto inclui módulos para operações sensíveis como manipulação de registro e captura de dados, então use com responsabilidade.

Posso adicionar novos módulos?

Sim! Basta criar uma classe Python seguindo o padrão dos módulos existentes (como DatabaseOps, MathOps, etc.) e registrá-la no CookieScriptVM. O sistema é extensível e novos módulos podem ser facilmente integrados.

Funciona em outros sistemas operacionais?

Atualmente otimizado para Windows, mas os módulos podem ser adaptados para Linux/Mac com modificações.

O que acontece se eu tentar usar uma variável indefinida?

O interpretador gera um erro de execução indicando que a variável não foi definida. Use try/catch para tratar esses erros.

Posso importar pacotes Python externos dinamicamente?

Sim, através do módulo python: python.importar_modulo(nome="pacote") permite importar e usar pacotes Python instalados.

Qual o impacto no desempenho da verificação de tipos?

A verificação de tipos é feita em tempo de execução e tem impacto mínimo em scripts típicos, mas pode ser notável em loops muito grandes.

Como lidar com timeouts de rede?

As requisições HTTP têm timeout padrão de 15 segundos. Para personalizar, use parâmetros adicionais quando disponível.

Há considerações de segurança que devo conhecer?

Sim, módulos como keylogger e registry manipulam dados sensíveis. Sempre use com responsabilidade e em ambientes controlados.
